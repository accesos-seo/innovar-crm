## 2. Slice 2 — Reasignación de visita + endurecimiento RLS

### 2.1 DB
- **024**: `ALTER TABLE visits` — el `visits_scheduled_via_check` ahora permite `'admin'`.
- **025**: RPC `public.assign_visit_to(p_visit_id uuid, p_new_visitor_id uuid) RETURNS visits` (SECURITY DEFINER). Hace:
  1. `get_my_role() IN ('admin','super_admin')` o `RAISE 42501`.
  2. Validar visit existente + visitante destino activo con rol válido.
  3. Liberar `availability_slots` del visitante anterior (`is_booked=false, task_id=null`).
  4. UPDATE `visits.visited_by` + `scheduled_via='admin'` (preserva `public_link` si así estaba).
  5. Trigger `visit_to_task_mirror` (UPDATE branch) actualiza `tasks.assigned_to` + pre-crea el nuevo slot. Trigger `sync_task_availability_booking` lo reserva.
- **025 hardening**: `REVOKE UPDATE ON public.visits FROM authenticated, anon` + `GRANT UPDATE (measurements, photos, status, modality, scheduled_at, duration_minutes, ...)` solo sobre 13 columnas operacionales. `visited_by`, `created_by`, `opportunity_id`, `public_token` **no son actualizables** desde SQL directo de cliente — la única vía es la RPC.

### 2.2 Frontend
- `src/hooks/agenda/useActiveVisitors.ts` (nuevo) — query profiles activos admin+comercial, staleTime 5min.
- `src/hooks/agenda/useAssignVisitTo.ts` (nuevo) — mutation a la RPC, mapea 42501 a "No tenés permisos…", invalida `appointments`/`visits`/`myVisitsToday`.
- `src/components/agenda/VisitOwnerPicker.tsx` (nuevo) — Select con `usePermissions().isAdmin`. Si no admin, muestra read-only con badge "Solo admin reasigna".
- `src/components/agenda/AppointmentDetailModal.tsx` (modificado) — monta `VisitOwnerPicker` cuando `appointment.appointment_type === 'visita_tecnica'`. Label "Visitante asignado" en lugar de "Comercial asignado" para visitas técnicas.

### 2.3 Smoke E2E SQL (corrido en QA opp)
- visit creada por `book_public_visit` → `visited_by=Alvaro` (gracias a S1)
- `assign_visit_to(visit, comercial)` → visit.visited_by=comercial, task.assigned_to=comercial, slot Alvaro liberado, slot comercial creado+reservado, scheduled_via=public_link (preservado).
- `UPDATE scheduled_via='comercial' THEN assign_visit_to(visit, alvaro)` → `scheduled_via='admin'` (promoción correcta por el CASE de la RPC).

Limitación: el caso "comercial intenta RPC → 42501" y "comercial intenta UPDATE directo de visited_by → permission denied" **no se prueba vía Management API** (sesión service_role bypassa RLS/grants). Se prueba con login real de un comercial.

## 3. Slice 3 — WhatsApp al admin + recordatorios 24h/2h

### 3.1 DB
- **026**: 3 funciones nuevas + 1 trigger + 2 crons:
  - `notify_visit_assigned_admin()` — trigger AFTER INSERT ON visits, encola `visit_assigned_admin_v1` al `visited_by` vía `fn_wa_enqueue_for_profile` (4 vars).
  - `enqueue_visit_reminders_24h_internal()` — cron `0 14 * * *` (09:00 Colombia), encola `visit_reminder_24h_internal_v1` al admin (5 vars). El lado cliente 24h ya lo cubre el cron existente `wa-recordatorio-24h-daily`.
  - `enqueue_visit_reminders_2h()` — cron `*/30 * * * *`, ventana de visitas en NOW()+1h45m..NOW()+2h15m, encola DUAL: `visit_reminder_2h_client_v1` al cliente + `visit_reminder_2h_internal_v1` al admin.
  - Idempotencia: dedup_key automático en `notification_queue` y `notifications`.

### 3.2 Edge Function

## 8. Riesgos conocidos

1. **Robert Anderson sigue activo** (D15): puede usar `assign_visit_to` si alguien entra con su cuenta. Mitigación pospuesta.
2. **Convención de timezone en `availability_slots`**: la 023b asume `start_time/end_time` están almacenados con valores extraídos de timestamps UTC. Si se cambiara la convención, todas las queries con `tstzrange &&` rompen silenciosamente. Documentado en [[feedback-innovar-availability-slots-source-of-truth]].
3. **`fn_queue_calendar_sync` impide DELETE directo de tasks**: para cleanup QA, hay que usar `UPDATE tasks SET status='cancelado'` (libera el slot vía trigger) en lugar de DELETE.
4. **TS errors pre-existentes** en otros archivos del repo (NewAppointmentModal, HardwareModule, MesonesModule, Pagos, etc) usan el patrón `string | null` mal tipado con shadcn Select de Base UI. No bloquean el build (Vite ignora type errors). Mis archivos S2-S5 pasan tsc limpios.
